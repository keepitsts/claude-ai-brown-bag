# TAC - Tactical Agentic Coding

## Tactic #1: Stop Coding

Your hands, and your mind are no longer the best tool for the job of writing code. Language models wrapped in the right agent architecture, running on super computers are vastly superior coders than you and I are. Engineering was never about writing code. So, stop coding and use the best tool for the job.

+ Daily Action
  + Resist the urge to type code manually
  + Go all in on Agentic Coding Tools
  + Don't code, write prompt and review
  + Communicate to your agents what you want built

## Tactic #2: Adopt Your Agent's Perspective

Your agent is brilliant, but blind. With every new session, it starts as a blank instance - ephemeral, no context, no memories. If you want your agent to perform like you would across your code base agentically, it needs your perspective. Your agent needs the information, tools, and resources you would use to solve the problem at hand. Whenever you launch you agent, ask yourself: With my agent's cour four (context, model, prompt, tools), is is possible to complete the task I've given it?

+ Daily Actions
  + Give your agent the context, model, prompt, and tools it needs
  + Ensure clear logging and standard output for agent visibility
  + Use types and consistent architecture to guide your agent
  + Write tests so your agent can self-validate its work
  + Equip your agent with what YOU would need to to the job

### The 12 Leverage Points of Agentic Coding

There are 12 leverage points of agentic coding you can use to multiply your agent's ability to execute on your behalf. These leverage points are the key to becoming an irreplaceable engineer.

+ In-Agent (Core Four)

  1) **Context** System prompts and conversation history -- Information & Perspective
  2) **Model** LLM selection and configuration -- Intelligence & Potential
  3) **Prompt** User instructions and queries -- Communication Method
  4) **Tools** Availble functions and capabilities -- Agent Capabilities

+ Through-Agent

  1) **Standard Out** Response from commands and logging -- Command Visibility
  2) **Types** Types, schemas, classes, errors, and structures -- Information Flow
  3) **Documentation** 3rd Party and internal -- Knowledge Base
  4) **Tests** Self-validating feedback loops -- Self-Validation
  5) **Architecture** Codebase structure -- Consistent Patterns
  6) **Plans** Detailed prompts for massive work -- Massive Agentic Prompts
  7) **Templates** Reusable embeddable agentic prompts -- Reusable Prompts
  8) **ADWs** AI Developer Workflows -- AI Developer Workflows

## Tactic #3: Template Your Engineering

When you template your engineering, you encode your engineering workflows and best practices into reusable, scalable units of agentic success. By encoding your problem-solving patterns into templates, you create a living library of engineering expertise that your agents can consistently execute with high precision across sets of problems.

With every template you create you're building system that enables your agents to solve problem sets and improve with every execution, turning your unique engineering wisdom into repeatable, autonomous workflows that scale across any codebase complexity

+ Daily Actions
  + Use meta-prompts to generate solutions for problem classes
  + Create specialized templates for engineering tasks (chores, bugs, features, refactors)
  + Build higher-order prompts that pass plans into plans
  + Review and refine templates based on execution outcomes
  + Document patterns and encode domain-specific problem-solving into templates
  + Convert successful prompts into meta-prompts that generate plans

## Tactic #4: Stay Out The Loop

Stay out the loop means building AFK (Away From Keyboard) agents that run while you're not there. Using PITER Framework (Prompt Input, Trigger, Environment, Review), you create out-of-loop agentic systems that can create AI Developer Workflows independently. This shifts you from in-loop prompting (back and forth with agents) to building autonomous systems that ship your work end-to-end, letting your product build itself while you focus on what matters most: Your Customers.

+ Daily Actions
  + Set up prompts inputs through GitHub issues, JIRA tickets, or Slack
  + Configure triggers using webhooks or scheduled tasks
  + Establish dedicated environments for safe agent execution
  + Build review systems through pull requests
  + Chain templates into ADWs for engineering automation

### The PITER Framework

The four elements of AFK agents (PITER):

  1. **Pi** Prompt input
  2. **T** Trigger
  3. **E** Environment
  4. **R** Review.

## Tactic #5: Always Add Feedback Loops

Your work is useless unless it's tested. Always add feedback loops enables your agents to act, validate, and correct in a continuous cycle until the job is done right. By teaching your agents to test through linters, unit tests, UI tests, and end-to-end validation, you create closed-loop systems where agents self-validate their work. This is the gift of generative AI - agents testing (and resolving) on your behalf at scales you could never achieve by hand.

+ Daily Actions
  + Add validation commands to every agentic prompt
  + Create closed-loop prompts: request, validate, resolve
  + Set up end-to-end tests for high ROI agent self-validation
  + Chain feedback mechanisms for comprehensive testing
  + Spending compute on testing will increase your confidence in your agent's work

## Tactic #6: One Agent, One Prompt, One Purpose

Massive context windows lead to distracted, confused agents.  One agent, one prompt, one purpose solves this by using specialized agents with focused prompts to achieve a single purpose well. This frees up the context window, lets agents focus on what matters most, and creates reproducible, improvable prompts you can commit to your codebase. Every step of engineering requires different information, tools, and context - honor this with dedicated agents for each purpose.

+ Daily Actions
  + Create dedicated agents for each workflow step
  + Keep prompts focused on single purposes
  + Free up context windows for complex problems
  + Commit and version control all agent prompts
  + Evaluate and improve individual agent performance

## Tactic #7: Target Zero-Touch Engineering

Zero-Touch Engineering is the third level of agentic coding beyond in-loop and out-loop, where your presence KPI drops from two touchpoints (prompt and review) to just one (prompt only). At this level, you witness something most engineers dismiss as impossible - a codebase that nearly runs itself.  Your agents ship end-to-end with such reliability that human review becomes a bottleneck, not a safety net, allowing work to flow directly from prompt to production.

+ Daily Actions
  + Build parallel agent pipelines using isolated environments
  + Scale from single agent to multi-agent worklfows
  + Create isolated environments for simultaneous execution
  + Go from PITER to PITE with one prompt agentic workflows
  + Build ADWs so great your review step becomes irrelevant
